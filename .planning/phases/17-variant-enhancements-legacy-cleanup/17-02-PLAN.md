# Plan 17-02 — Admin Variant Editor Extensions + Reactivity (Wave 1)

## Goal

Extend `src/components/admin/variant-editor.tsx` and `src/actions/variants.ts` to support all Phase 17 admin surfaces, and rewire `src/actions/shipping-quote.ts` to consume per-variant weight (AD-08):

### 1. New server actions in `src/actions/variants.ts`

#### 1a. `uploadVariantImage(variantId, formData)` (AD-02, AD-06 Pattern A)

```ts
export async function uploadVariantImage(
  variantId: string,
  formData: FormData,
): Promise<ActionResult<{ imageUrl: string }>> {
  await requireAdmin();
  // 1. Resolve productId from variant
  const [v] = await db
    .select({ productId: productVariants.productId, oldUrl: productVariants.imageUrl })
    .from(productVariants)
    .where(eq(productVariants.id, variantId))
    .limit(1);
  if (!v) return { error: "Variant not found" };

  // 2. writeUpload with bucket=productId (sanitized)
  const file = formData.get("file");
  if (!(file instanceof File)) return { error: "No file provided" };
  let newUrl: string;
  try {
    newUrl = await writeUpload(v.productId, file);
  } catch (e) {
    return { error: e instanceof Error ? e.message : "Upload failed" };
  }

  // 3. DB update
  try {
    await db
      .update(productVariants)
      .set({ imageUrl: newUrl })
      .where(eq(productVariants.id, variantId));
  } catch (e) {
    // Rollback filesystem on DB failure (T-17-02b-upload-orphan)
    await deleteUpload(newUrl).catch(() => {});
    return { error: "Failed to persist image URL" };
  }

  // 4. Best-effort delete of prior image
  if (v.oldUrl) {
    await deleteUpload(v.oldUrl).catch(() => {});
  }

  revalidatePath(`/admin/products/${v.productId}/variants`);
  return { success: true, data: { imageUrl: newUrl } };
}
```

Import `writeUpload`, `deleteUpload` from `@/lib/storage`.

#### 1d. `bulkUpdateVariants(productId, variantIds, op)` (AD-03, AD-06 Pattern B)

```ts
type BulkOp =
  | { kind: "set-price"; value: string }          // MYR
  | { kind: "multiply-price"; percent: number }   // e.g. 110 = +10%
  | { kind: "add-price"; delta: string }          // e.g. "-5.00"
  | { kind: "set-sale-price"; value: string | null }
  | { kind: "set-active"; inStock: boolean }
  | { kind: "delete" };
```

Server action signature:

```ts
export async function bulkUpdateVariants(
  productId: string,
  variantIds: string[],
  op: BulkOp,
): Promise<ActionResult<{ affected: number }>>
```

Validation (critical — threat mitigations):
- Every variantId belongs to productId (T-17-06-bulk-scope-leak): `SELECT id FROM product_variants WHERE id IN (?) AND product_id = ?`; any mismatch → reject the whole op.
- `multiply-price` with `percent <= 0` or `!isFinite(percent)` → reject (T-17-01c).
- After op, resultant price must be `>= 0.01` (T-17-01b) for every affected row; if any would go below, reject the whole op with `error: "Bulk op would set variant X price below RM 0.01"`.
- `set-sale-price`: if not null, must be `< current price` for every affected row (T-17-01-price-tampering). Reject whole op if any row fails.

Transaction: wrap all row updates in `db.transaction(async (tx) => { ... })` so a failure rolls back everything.

Return count affected.

const handleUpdateVariantField = (variantId: string, field: string, value: unknown) => {
  const prev = variants; // snapshot for rollback
  setVariants((p) =>
    p.map((v) => (v.id === variantId ? { ...v, [field]: value } : v)),
  );
  startTransition(async () => {
    const result = await updateVariant(variantId, { [field]: value });
    if ("error" in result) {
      setVariants(prev); // rollback
      showToast(result.error, "error");

  });
};

### 6. Rewire Delyva quote to use variant weight (AD-08)

Edit `src/actions/shipping-quote.ts`:

- Extend `CartItemForQuote` type — `variantId: string` becomes MANDATORY (not optional). Since Phase 16 AD-05 cart v2 stores variantId for every line, every caller already has it.
- In `quoteForCart`:
  - Batch-fetch `productVariants.weightG` for all variantIds in the cart via a single `inArray(productVariants.id, variantIds)` query. Build `variantWeights: Map<string, number | null>` keyed by variantId.
  - Keep the existing `products.shippingWeightKg` batch fetch as the fallback tier.
  - Per line item, resolve weight in this order:
    1. `variant.weightG` (grams → kg via `/1000`) when non-null
    2. `product.shippingWeightKg` (already kg) when non-null
    3. `cfg.defaultWeightKg` (existing fallback, default 0.5)
    4. Emit `console.warn(...)` with `{ variantId, productId }` when tier 3 is used — i.e. both the variant and its product have no weight at all.
  - Multiply by line `quantity`; sum across lines; pass the total to `delyvaApi.quote({ weight: { unit: "kg", value: round2(totalWeight) }, ... })` unchanged.

Edit all producers / callers of `CartItemForQuote`:

- `src/app/(store)/checkout/page.tsx` (or wherever checkout builds the quote payload) — forward `variantId` from cart lines into the `CartItemForQuote[]` payload.
- `src/stores/cart-store.ts` + hydration helpers — the cart already stores `variantId` for every line (Phase 16-05 AD-05); any adapter that maps cart → quote payload must include it.
- Audit surface: grep for `CartItemForQuote` and `quoteForCart(` to confirm all call sites are updated — compile will fail any missed producer because `variantId` is now mandatory.

Threat mitigation (T-17-09-weight-spoofing per 17-PLAN): server re-fetches `weightG` by variantId on every quote — never trusts client-supplied weight values.

Acceptance for this task:
- Given two variants S (weightG=50) and L (weightG=300) of the same SKU family, a cart with 1×S quotes a lower Delyva `value` than a cart with 1×L to the same destination postcode.
- NULLing the `weight_g` on L via admin editor causes the next quote to use `products.shippingWeightKg × 1000` instead (observable delta).
