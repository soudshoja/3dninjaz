<objective>
Ship the **storefront PDP for made-to-order products**. Branch the existing `<ProductDetail>` (currently `src/components/store/product-detail.tsx`) on `productType` — add ≤5 additive lines that route to `<ConfigurableProductView>` when `productType === "configurable"`. The new view renders:
- `<ConfigurableImageGallery>` — hero + thumbstrip; default to admin's display image; first text input or colour pick auto-swaps hero to live preview ("Yours" thumbnail); user can flip back via thumbstrip click.
- `<ConfiguratorForm>` — one input per `product_config_fields` row, dispatching by fieldType.
- `<KeychainPreview>` — generic SVG name strip rendering current text in chosen base+letter colours; works for any text-driven product.
- **Tier-lookup price meter** — uses `lookupTierPrice(priceTiers, unitFieldValue)`. Disables Add-to-bag when out of table.

Phase 17 PDP variant flow stays UNTOUCHED for stocked products (D-14 — ≤5 lines added to product-detail.tsx).

Purpose: REQ-7. Without this customers cannot fill the form or see live preview. The cart payload (Plan 19-08) is what they "Add to bag" submits.
Output: 1 modified PDP entry + 4 new components + 1 new server-side hydration helper.
</objective>

    ```typescript
    function buildSummary(fields, values, price) {
      const parts: string[] = [];
      for (const f of fields) {
        const v = values[f.id] ?? "";
        if (!v) continue;
        if (f.fieldType === "text") parts.push(`"${v}" (${v.length} ${f.label.toLowerCase()})`);
        else if (f.fieldType === "colour") {
          const c = f.resolvedColours?.find((x) => x.id === v);
          if (c) parts.push(`${c.name} ${f.label.toLowerCase()}`);
        } else parts.push(`${f.label}: ${v}`);
      }
      return parts.join(" · ");
    }
    ```
