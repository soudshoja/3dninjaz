<objective>
Extend Phase 7's image pipeline (`src/lib/image-pipeline.ts`) to **emit additional WebP/AVIF widths suited to PDP heroes** AND **store an admin-editable caption per image** at the database level. Wire it into:
- **Admin product-form upload UI** — caption text input below each image thumbnail.
- **Configurable product PDP gallery** (`<ConfigurableImageGallery>` from Plan 19-06) — renders `<picture>` with `<source srcset>` for AVIF + WebP + JPEG fallback at all available widths; hero swap to live-preview keeps existing behavior.
- **Stocked product PDP** — UNCHANGED. Phase 7's existing `<ResponsiveProductImage>` already does the picture+srcset rendering for stocked products. We extend the pipeline output (more widths) but the existing reads adapt automatically because manifest.json schema is forward-compatible (consumers iterate `manifest.widths`).

**Per CONTEXT.md D-05:** images JSON shape widens from `string[]` to `Array<{ url: string; caption?: string; alt?: string }>`. The Sharp pipeline OUTPUT (manifest.json under each image dir) is unchanged in shape — only widths array gets `[480, 960, 1440]` ADDED to existing `[400, 800, 1600]`, totaling 6 widths. That decision is reversed in this plan: per CONTEXT.md D-05 explicit widths are `480, 960, 1440`. **We adopt the explicit D-05 list and leave the existing 400/800/1600 widths as a backwards-compat read path** — see `ensureImagesV2` for the read-side logic.

**Per CONTEXT.md D-05 (image gallery: schema + Sharp pipeline):**
- products.images JSON column: widen shape via `ensureImagesV2(raw)` helper (already exists from Plan 19-02 — confirm and extend if needed) that handles both legacy `string[]` AND new `Array<{ url, caption, alt }>` entries
- Sharp pipeline extension: on each upload, generate WebP + AVIF at 480/960/1440 widths
- PDP `<picture>` consumes the manifest

**Per CONTEXT.md D-14:** variant code path receives ZERO additional lines. The existing `<ResponsiveProductImage>` continues to consume the same `manifest.json` shape; only the `widths` array entries grow. New widths are additive — old stocked-product manifests with `[400, 800, 1600]` continue to work.

Purpose: REQ-6 — unlimited images, captions, multi-resolution srcset. LCP target: ≤ 200 KB at 1440-wide viewport. Without this plan the configurable PDP renders display image at full original resolution (3+ MB on mobile), tanking LCP.
Output: pipeline extension + caption schema widening + new PDP picture component for configurable products + admin caption UI + read-side `ensureImagesV2` consumer wiring.
</objective>

         ```typescript
         export type ImageEntryV2 = {
           url: string;
           caption?: string | null;
           alt?: string | null;
         };
         export function ensureImagesV2(raw: unknown): ImageEntryV2[] {
           if (raw == null) return [];
           let parsed: unknown = raw;
           if (typeof raw === "string") {
             try { parsed = JSON.parse(raw); } catch { return []; }
           }
           if (!Array.isArray(parsed)) return [];
           return parsed.map((entry) => {
             if (typeof entry === "string") return { url: entry, caption: null, alt: null };
             if (entry && typeof entry === "object" && typeof (entry as { url?: unknown }).url === "string") {
               const o = entry as { url: string; caption?: unknown; alt?: unknown };
               return {
                 url: o.url,
                 caption: typeof o.caption === "string" ? o.caption : null,
                 alt: typeof o.alt === "string" ? o.alt : null,
               };
             }
             return null;
           }).filter((x): x is ImageEntryV2 => x !== null);
         }
         ```
