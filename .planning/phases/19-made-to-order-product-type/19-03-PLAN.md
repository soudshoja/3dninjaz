<objective>
Wire the **Stocked vs Made-to-Order** product-type radio at the top of the product form (D-06). Two-card radio with descriptions; persists `productType`. On the Edit page the radio is **disabled** with a flip-block message when the product already has variants OR config fields attached. Server action `updateProductType(productId, newType)` enforces the same rule server-side. This plan also creates `src/actions/configurator.ts` as the home file for the Wave 2/3 server actions (only `updateProductType` lands here in this plan; rest land in Plan 19-04).

Purpose: REQ-3 — admin chooses type before name/description. Without this gate the configurator builder has nothing to branch on. REQ-9 backwards compat — flip-block prevents accidental data destruction on a 30-variant product.

Output: New radio component + updateProductType action + product form integration + 2 page wirings. Stocked-product variant editor untouched.
</objective>

    Pseudocode:
    ```typescript
    "use server";
    import { randomUUID } from "node:crypto";
    import { db } from "@/lib/db";
    import { products, productVariants, productConfigFields } from "@/lib/db/schema";
    import { eq, and } from "drizzle-orm";
    import { revalidatePath } from "next/cache";
    import { requireAdmin } from "@/lib/auth-helpers";

    export async function updateProductType(productId: string, newType: "stocked"|"configurable") {
      await requireAdmin();              // FIRST await — CVE-2025-29927
      const [row] = await db.select({ id: products.id, type: products.productType })
        .from(products).where(eq(products.id, productId)).limit(1);
      if (!row) return { ok: false as const, error: "Product not found" } as const;
      if (row.type === newType) return { ok: true as const };  // no-op fast path

      // Block if attached data of the OTHER type exists (defense — UI also blocks).
      // Going stocked -> configurable: must have zero variants.
      // Going configurable -> stocked: must have zero config fields.
      if (newType === "configurable") {
        const [{ count }] = await db.execute(
          /* SELECT COUNT(*) AS count FROM product_variants WHERE product_id=? */
          // use drizzle: db.select({count: sql<number>`COUNT(*)`}).from(productVariants).where(eq(productVariants.productId, productId))
        );
        if (count > 0) return { ok: false as const, error: "Cannot change product type with attached variants" } as const;
      } else {
        const [{ count }] = await db.select({...}).from(productConfigFields).where(eq(productConfigFields.productId, productId));
        if (count > 0) return { ok: false as const, error: "Cannot change product type with attached config fields" } as const;
      }

      await db.update(products).set({ productType: newType }).where(eq(products.id, productId));
      revalidatePath("/admin/products");
      revalidatePath(`/admin/products/${productId}/edit`);
      return { ok: true as const };
    }
    ```

    Use Drizzle `sql` helper for COUNT(*) — see existing examples in `src/actions/wishlist.ts` or similar count queries.
