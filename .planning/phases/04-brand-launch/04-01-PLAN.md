**Part B — `src/components/seo/json-ld.tsx`**

Create a typed server component that emits a schema.org JSON payload as a script tag WITHOUT using `dangerouslySetInnerHTML`. Use React's official JSX-safe pattern for JSON-LD: render the JSON string as the script's `children`, wrapped in a `<script type="application/ld+json">` element. React will safely escape the text-content children automatically.

Signature:
```ts
type JsonLdData = Record<string, unknown>;
export function JsonLd({ data }: { data: JsonLdData }) {
  return (
    <script
      type="application/ld+json"
      // React auto-escapes children text nodes; no dangerouslySetInnerHTML needed.
    >
      {JSON.stringify(data)}
    </script>
  );
}
```

This pattern is used by Next.js's own documentation examples for JSON-LD and avoids the XSS risk surface entirely because the content is produced via `JSON.stringify` on a fully server-controlled object.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Server → public `<head>` | Metadata module values reach every browser; stale/leaky content here is a brand/SEO defect, not a security one |
| Public file routes | `src/app/icon*.png`, `apple-icon.png`, `favicon.ico`, `public/og-default.png` served to any client |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01-01 | I (Information Disclosure) | `SITE.legalName` / `SITE.description` | mitigate | Do NOT include the legal entity name, business address, or registration number in Task 1 — those are Plan 04-02 concerns. Only public trading name + tagline in site-metadata.ts. |
| T-04-01-02 | T (Tampering) | JSON-LD scripts in layout.tsx | mitigate | Use the `JsonLd` component from Task 1 which renders JSON-LD as React children (auto-escaped) instead of `dangerouslySetInnerHTML`. Content is fully server-controlled via SITE constant — no user input flows into JSON-LD. |
| T-04-01-03 | S (Spoofing) | OG image `public/og-default.png` | accept | Copying `public/logo.png` is a known brand asset under the project's control. No external fetch, no user upload path. |
| T-04-01-04 | D (Denial of Service) | Favicon serving | accept | Next.js file-based icons are statically served; no runtime cost beyond static asset delivery. |
| T-04-01-05 | I (Information Disclosure) | Accidental `robots: noindex` leak | mitigate | Task 3 hard-codes `robots: { index: true, follow: true }`. Plan 04-04 separately handles removal of the coming-soon `<meta name="robots" content="noindex">`. |
</threat_model>
