<task type="auto">
  <name>Task 2: Image audit — enforce width/height, priority, sizes across all customer-facing Next.js Image usages</name>
  <files>src/components/brand/logo.tsx, src/components/store/product-card.tsx, src/app/(store)/layout.tsx (already touched), any other file containing next/image imports under src/</files>
  <action>
Run a repo-wide scan for `next/image` usages and verify each one conforms to a clear intrinsic-sizing strategy. Acceptable patterns:
- Fixed dimensions: explicit `width={N}` and `height={N}` props
- Responsive with container: `fill` + `sizes="(max-width: 768px) 100vw, 50vw"` pattern

Unacceptable patterns to fix:
- Missing both width/height and fill
- `fill` without `sizes` (blocks LCP optimization and triggers a Next.js dev warning)
- `priority` applied to below-the-fold images (burns bandwidth; removes implicit lazy)

Scan command:
```bash
grep -rn "from \"next/image\"" src/ --include="*.tsx"
grep -rn "<Image " src/ --include="*.tsx"
```

For each hit, read the file and determine:
1. Does this image have explicit width/height or fill + sizes? -> if no, fix.
2. Is this image above the fold on first render? -> if yes, `priority`; if no, no priority (Next auto-lazy-loads).
3. For product card thumbnails: add `sizes="(max-width: 640px) 50vw, (max-width: 1024px) 33vw, 25vw"` (matches the 2/3/4-column grid).
4. For hero logo/banner: `priority` is correct.

Create a brief note in `.planning/phases/04-brand-launch/RESPONSIVE-AUDIT.md` with a list of files changed and the rationale. Do NOT create a separate commit for each fix — batch all image fixes into this task.

If a file has no fixes required, still log it in the audit as "Reviewed: OK."
  </action>
  <verify>
    <automated>bash -c 'fails=0; for f in $(grep -rl "next/image" src/ --include="*.tsx"); do if grep -q "<Image " "$f" && ! grep -qE "width=|height=|fill" "$f"; then echo "MISSING dims: $f"; fails=$((fails+1)); fi; if grep -q "fill" "$f" && ! grep -q "sizes=" "$f"; then echo "MISSING sizes (with fill): $f"; fails=$((fails+1)); fi; done; exit $fails'</automated>
  </verify>
  <done>
- Every `<Image>` in src/ has either explicit width/height OR fill+sizes
- Hero/above-the-fold images use priority; below-the-fold do not
- RESPONSIVE-AUDIT.md lists every file reviewed and the action taken
  </done>
