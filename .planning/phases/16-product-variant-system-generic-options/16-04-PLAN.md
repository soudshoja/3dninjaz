# Plan 16-04 — Storefront PDP Variant Selector (Wave 2)

**Depends on:** 16-02
**Parallel with:** 16-03

## Goal

Product detail page variant selector that handles 1..3 options dynamically. Renders swatches for Color (when swatchHex set), pill buttons otherwise. Updates selected variant's price/stock/image live.

## Tasks

1. **Client component `src/components/store/variant-selector.tsx`**
   - Props: `{ options, variants, onVariantChange }`
   - State: `selectedValueIds: (string | null)[]` (3 slots)
   - For each option, render group:
     - If option has ≥1 value with `swatchHex` set → swatch buttons (colored circle + tooltip label)
     - Else → pill buttons (rounded-full text button)
   - Unavailable combos (no matching variant OR variant inStock=false) render disabled state
   - On selection change, compute `selectedVariant = findVariantByValueIds(variants, selectedValueIds)` and fire `onVariantChange`
   - Default selection: first available variant (by position)
   - Mobile-first: 48px tap targets per Phase 2 D-04

2. **PDP page edits — `src/app/(store)/product/[slug]/page.tsx`** (or the client-side product detail client component)
   - Replace existing size selector with `<VariantSelector options={hydratedOptions} variants={hydratedVariants} onVariantChange={setSelectedVariant} />`
   - Price display reads `selectedVariant.price`
   - Stock display reads `selectedVariant.inStock` + `selectedVariant.stock` (if trackStock)
   - Image gallery: if `selectedVariant.imageUrl` set, prepend to gallery; else use product's image array
   - "Add to bag" button fires cart action with `{ variantId: selectedVariant.id, quantity }`

3. **PDP shell — size guide + material info unchanged**
   - Size guide section only renders if product has a "Size" option (detect by option name)
   - Keeps Phase 2 D-13 size guide UI for size-option products

4. **Legacy fallback path**
   - If `hydratedVariants` missing (shouldn't happen post-backfill but defensive), render legacy `<SizeSelector>` via `legacyVariantToHydrated` mapping
   - Console warn in dev: `[phase16] product %s falling back to legacy variant path`

5. **Visual styling**
   - Swatches: 40px circle, 2px border on selected, 1px border on unselected, disabled = 40% opacity + line-through
   - Pills: existing `btn-variant` styles if defined; else Tailwind `rounded-full border px-4 py-2 text-sm data-[active]:bg-ink data-[active]:text-cream`
   - Label above each option group: option.name in bold
   - Selected value displayed inline: "Color: Red"
