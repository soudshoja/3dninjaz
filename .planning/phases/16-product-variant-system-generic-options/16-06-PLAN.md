# Plan 16-06 — Inventory / Cost Breakdown / CSV Import Adapted (Wave 3)

**Depends on:** 16-05

## Goal

All existing per-variant admin tooling (inventory page, cost breakdown UI, CSV import) understands arbitrary variant shapes — not just S/M/L.

## Tasks

1. **Inventory page `/admin/inventory`**
   - Replace hardcoded S/M/L column layout with per-variant rows: `Product | Variant Label | Stock | Track Stock | In Stock | Low Threshold`
   - Filter: products with any out-of-stock variant
   - Sort by product then by variant position
   - Inline edits hit the Phase 16-03 `updateVariant` action

2. **Cost breakdown UI — admin product form**
   - Cost breakdown fields (filamentGrams, printTimeHours, laborMinutes, etc. — Phase 14) live per-variant
   - Move inputs from product form into variant editor matrix (new column or expanded row in 16-03 table)
   - Store settings defaults (Phase 14) unchanged
   - Live cost preview per variant

3. **CSV import `/admin/products/import`**
   - New column schema:
     - Existing columns (name, description, category, images) unchanged
     - REMOVE `price_s`, `price_m`, `price_l` columns
     - ADD columns: `option1_name`, `option1_values` (pipe-separated e.g., `S|M|L`), `option2_name`, `option2_values`, `option3_name`, `option3_values`
     - ADD columns: `variant_combos` (pipe-separated `S/Red,M/Red,L/Red|S/Blue,M/Blue,L/Blue`) with `variant_prices` and `variant_stock` aligned
   - On import: create product → create options + values → cartesian-expand or explicit-match from variant_combos → insert variants with prices
   - Template CSV generator updated to new header layout
   - Preview step shows expanded variant table before commit
   - Single transaction; rollback on any failure (per Phase 5 05-05 pattern)

4. **REMOVE legacy size-based hardcoded paths**
   - Grep for `"S"|"M"|"L"` enum references in admin code; replace with variant-label reads
   - Grep for `variants[0].size` or similar; replace with `hydratedVariants[0].label`
   - Code review pass: any path still asserting size=`S/M/L` is a bug
