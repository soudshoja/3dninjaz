# Plan 16-02 — Backfill + Dual-Read Helpers (Wave 1)

**Depends on:** 16-01
**Unblocks:** 16-03, 16-04

## Goal

Backfill every existing product with an implicit "Size" option and S/M/L values. Wire existing variants' `option1_value_id` + `label_cache`. Add `src/lib/variants.ts` helpers that any consumer can call — dual-reads new + legacy columns transparently.

## Tasks

1. **Backfill script `scripts/phase16-backfill.cjs`**
   - For each product in `products`:
     - Insert one row into `product_options` with `name="Size"`, `position=1` (idempotent via SELECT-first)
     - Insert three rows into `product_option_values` with values `Small`, `Medium`, `Large` at positions 1/2/3 (idempotent)
   - For each row in `product_variants` where `option1_value_id IS NULL`:
     - Look up the option_value matching the variant's `size` ("S"→"Small", "M"→"Medium", "L"→"Large")
     - `UPDATE product_variants SET option1_value_id=?, label_cache=? WHERE id=?`
   - Idempotent: re-running = zero updates
   - Report: products processed, variants updated, errors

2. **Helper module `src/lib/variants.ts`**
   - `export async function hydrateProductVariants(productId)`: returns shape
     ```
     { options: [{ id, name, position, values: [{ id, value, position, swatchHex? }] }], 
       variants: [{ id, price, stock, inStock, sku, imageUrl, label, optionValueIds: [v1?, v2?, v3?] }] }
     ```
     Fetch parent options, then values, then variants in 3 queries (manual hydration per MariaDB no-LATERAL rule). Map `option1_value_id` → option value for label composition.
   - `export function composeVariantLabel(optionValues)`: joins values with " / " — e.g., `["Small", "Red"]` → `"Small / Red"`
   - `export async function findVariantByOptions(productId, valueIds[])`: SELECT variant where `option1_value_id=?` AND `option2_value_id=?` AND `option3_value_id=?` (NULL-safe). Returns variant or null.
   - `export function legacyVariantToHydrated(variantRow)`: dual-read fallback — if `option1_value_id IS NULL` but `size` column is set, return synthetic `{ label: variantRow.size, ... }` so old reads don't break during rollout

3. **Catalog integration — `src/lib/catalog.ts`**
   - `getProduct(slug)`: extend return type with `options` and `hydratedVariants` fields; keep existing `variants` array (with `size`) for backward compat during rollout
   - `getProducts()` list: same shape addition, but hydration is ONLY for fetched products (no N+1) — batch fetch options and values keyed by productId

4. **Unit tests `src/lib/__tests__/variants.test.ts`** (if test infra exists; else skip with note)
   - `composeVariantLabel(["Small"])` → `"Small"`
   - `composeVariantLabel(["Small", "Red"])` → `"Small / Red"`
   - `legacyVariantToHydrated({ size: "M", price: "10.00" })` → label `"M"` (pre-migration fallback)

5. **Run backfill**
   - `node scripts/phase16-backfill.cjs`
   - Verify: `SELECT COUNT(*) FROM product_variants WHERE option1_value_id IS NULL;` returns 0
