# Plan 16-01 — Schema + Migration Applicator (Wave 1)

**Depends on:** — (first plan)
**Unblocks:** 16-02, 16-03, 16-04

## Goal

Land new schema additions (product_options, product_option_values, variant columns, order_items.variant_label) via raw-SQL migration applicator. Update `src/lib/db/schema.ts` with Drizzle definitions. Add Zod validators. Register VAR-01..VAR-06 requirements.

## Tasks

1. **Schema edits in `src/lib/db/schema.ts`**
   - Add `productOptions` table: `(id uuid pk, product_id fk, name varchar(64), position int)` — UNIQUE(product_id, position), UNIQUE(product_id, name)
   - Add `productOptionValues` table: `(id uuid pk, option_id fk, value varchar(64), position int, swatch_hex char(7) NULL)` — UNIQUE(option_id, value)
   - Extend `productVariants`:
     - `option1_value_id varchar(36) NULL references productOptionValues(id) ON DELETE RESTRICT`
     - `option2_value_id varchar(36) NULL references productOptionValues(id) ON DELETE RESTRICT`
     - `option3_value_id varchar(36) NULL references productOptionValues(id) ON DELETE RESTRICT`
     - `sku varchar(64) NULL`
     - `image_url text NULL` (simple — single image per variant)
     - `label_cache varchar(200) NULL`
     - `position int NOT NULL DEFAULT 0`
     - KEEP `size` column for now (dual-read window)
   - Extend `orderItems`:
     - `variant_label varchar(200) NULL`
     - KEEP `size` column for historical orders
   - Add relations: `productOptionsRelations`, `productOptionValuesRelations`; extend `productsRelations` with `options: many(productOptions)`; extend `productVariantsRelations` with `option1Value`, `option2Value`, `option3Value` (one) — use explicit `relationName` per FK

2. **Raw-SQL applicator `scripts/phase16-migrate.cjs`**
   - Pattern after `scripts/phase6-migrate.cjs`, `scripts/phase5-migrate.cjs`, `scripts/phase7-migrate.cjs`
   - Use `mysql2/promise`, load from `DATABASE_URL` env
   - Idempotent DDL: `CREATE TABLE IF NOT EXISTS` + check `information_schema.COLUMNS` before each `ALTER TABLE ... ADD COLUMN`
   - Charset `latin1` to match existing tables (FK constraint requirement documented in Phase 6 notes)
   - Verify with `SHOW CREATE TABLE` after each DDL
   - Exit code 0 on clean success, 1 on any error with full error message

3. **Zod validators in `src/lib/validators.ts`**
   - `productOptionSchema`: `{ name: string.min(1).max(64), position: number.int().min(1).max(3) }`
   - `productOptionValueSchema`: `{ value: string.min(1).max(64), position: number.int().min(0), swatchHex: string.regex(/^#[0-9a-f]{6}$/i).optional() }`
   - `productVariantSchema` extension: add optional `option1ValueId`, `option2ValueId`, `option3ValueId`, `sku` (max 64), `imageUrl` (url().optional()), `position`

4. **Requirements registration**
   - Edit `.planning/REQUIREMENTS.md`, append section "Phase 16 — Variant System":
     - VAR-01: Admin defines 1..3 options per product
     - VAR-02: Cartesian variant matrix auto-generated
     - VAR-03: Existing products auto-migrate via backfill
     - VAR-04: PDP renders N-option selector with price/stock/image updates
     - VAR-05: Cart/checkout/orders reference variant_id + label
     - VAR-06: Parts-based product end-to-end PayPal flow

5. **Run migration**
   - `node scripts/phase16-migrate.cjs` — apply to the cPanel MariaDB via DATABASE_URL
   - Verify via direct SQL: `DESCRIBE product_options; DESCRIBE product_option_values; DESCRIBE product_variants;`
