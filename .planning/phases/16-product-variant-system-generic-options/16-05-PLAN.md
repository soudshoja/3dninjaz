# Plan 16-05 — Cart v2 + Checkout/Orders Rewire (Wave 3)

**Depends on:** 16-03, 16-04

## Goal

Cart store references `variantId` only. Checkout server actions hydrate label + price from DB at capture time. Orders persist `variant_label` snapshot alongside `variantId`. PayPal line items use composed label.

## Tasks

1. **Zustand cart store — `src/lib/cart-store.ts`**
   - Bump localStorage key to `print-ninjaz-cart-v2` (auto-clears v1 carts on first load)
   - Line item shape: `{ variantId, quantity }` — no productId, no size, no price, no label
   - Hydration helper `hydrateCartItems(items)` — batch SELECTs variants + products + options/values → returns display-ready `{ variantId, quantity, productSlug, productName, productImage, variantLabel, unitPrice, inStock, available }`
   - Drawer + /bag page consume hydrated shape
   - `addToCart({ variantId, quantity })` — legacy callers passing `{productId, size}` shimmed via `resolveVariantFromLegacy` helper during rollout (logs deprecation warning in dev)

2. **Checkout server action — `src/actions/checkout.ts` or `src/actions/orders.ts`**
   - Re-hydrate cart items from DB at checkout (price, label, stock check)
   - If any variant missing or inStock=false → return error + user-facing message "Item {label} is no longer available"
   - Stock reservation logic (Phase 13 pattern) keyed on variantId

3. **Orders persistence — `src/actions/orders.ts`**
   - `createOrderFromCart(userId, addressId, items[])`:
     - Compose `variant_label` at insert time: `composeVariantLabel(hydratedValues)`
     - `order_items.variant_id` already exists — populate correctly
     - `order_items.variant_label` = composed string
     - `order_items.size` = first option value's text for legacy rendering (safe fallback) OR leave NULL if no Size option → keep backwards-compatible with legacy invoice/email renderers during rollout

4. **PayPal line items — `src/lib/paypal.ts`**
   - `buildLineItems(orderItems)`: name = `${productName} — ${variant_label}` (truncated to PayPal's 127 char limit)
   - Price per line = unit_price; quantity = quantity
   - Existing signature-verified webhook unchanged (it references orderId not variantId)

5. **Order rendering**
   - `/orders/[id]` customer page: render `variant_label` if set, fallback to `size`
   - `/admin/orders/[id]` admin page: same
   - Invoice PDF (`@react-pdf/renderer`): same fallback
   - Order confirmation email template: variable `{item.variantLabel}` with fallback `{item.size}` — update via admin email template editor if needed

6. **Back-compat shim for in-flight carts**
   - On first load under v2, read legacy v1 key, attempt `resolveVariantFromLegacy` per item, migrate to v2 shape, delete v1 key
   - If any item fails to resolve, drop silently (log warning in dev)
