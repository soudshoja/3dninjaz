<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Admin browser -> Cloudinary | Image upload from admin browser to Cloudinary CDN via upload widget |
| Admin form -> Server action | Product form data submitted to createProduct/updateProduct |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | Image upload | mitigate | Cloudinary widget configured with clientAllowedFormats (jpg/png/webp), maxFileSize 5MB, maxFiles 5. Admin must be authenticated to access the form. Upload preset restricts resource types in Cloudinary dashboard. |
| T-04-02 | Tampering | Product form data | mitigate | All data validated server-side via Zod productSchema before database write. Client validation is convenience only. Images array validated as URL strings, max 5. |
| T-04-03 | Elevation of Privilege | Product create/edit | mitigate | createProduct and updateProduct both call requireAdmin() before any DB operation (CVE-2025-29927 mitigation from Plan 03). |
| T-04-04 | Denial of Service | Large image uploads | accept | Cloudinary enforces 5MB per file limit. Free tier has 10GB total storage. Acceptable risk for admin-only uploads. |
| T-04-05 | Information Disclosure | Cloudinary upload preset | mitigate | Upload preset is unsigned but restricted to image types and max file size in Cloudinary dashboard settings. Admin-only access to the upload form. Monitor Cloudinary dashboard for unexpected uploads. |
</threat_model>
