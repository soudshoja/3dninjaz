```typescript
// Seed script to create the first admin user
// Run with: npm run seed:admin (which runs npx tsx scripts/seed-admin.ts)
import { auth } from "../src/lib/auth";

async function seedAdmin() {
  const email = process.env.ADMIN_EMAIL || "admin@3dninjaz.com";
  const password = process.env.ADMIN_PASSWORD || "changeme123";
  const name = "3D Ninjaz Admin";

  try {
    const result = await auth.api.createUser({
      body: {
        email,
        password,
        name,
        role: "admin",
      },
    });
    console.log("Admin user created successfully:", result.user.email);
  } catch (error: any) {
    if (error.message?.includes("already exists") || error.status === 422) {
      console.log("Admin user already exists, skipping.");
    } else {
      console.error("Failed to create admin:", error);
      process.exit(1);
    }
  }
}

seedAdmin();
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client form -> Auth API | Registration and login data from untrusted browser |
| Password reset link -> Reset form | Token in URL could be intercepted or expired |
| Seed script -> Database | Elevated privilege operation run locally |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | Login form | mitigate | Better Auth handles rate limiting and password verification. No custom auth logic. |
| T-02-02 | Information Disclosure | Forgot password | mitigate | Same response message whether email exists or not -- prevents email enumeration. |
| T-02-03 | Repudiation | PDPA consent | mitigate | pdpaConsentAt is a server-side ISO timestamp set at submission time, not a client boolean. Stored in user record. |
| T-02-04 | Tampering | Registration form | mitigate | Client validates password match/length; server (Better Auth) enforces password requirements independently. Role always defaults to "customer" -- cannot be set via registration. |
| T-02-05 | Elevation of Privilege | Seed script | accept | Seed script runs locally with direct DB access. Only developers with .env.local can run it. Not exposed as an API endpoint. |
| T-02-06 | Spoofing | Password reset token | mitigate | Better Auth generates cryptographic tokens with expiry. Tokens are single-use. |
</threat_model>
