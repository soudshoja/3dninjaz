2. Create `src/lib/db/index.ts` — Drizzle client singleton backed by a mysql2 pool. Reuse the pool across hot-reloads via a `globalThis` handle so dev reloads don't exhaust the cPanel MySQL connection cap:
```typescript
import { drizzle } from "drizzle-orm/mysql2";
import mysql from "mysql2/promise";
import * as schema from "./schema";

declare global {
  // eslint-disable-next-line no-var
  var __mysqlPool: mysql.Pool | undefined;
}

function buildPool(): mysql.Pool {
  const url = process.env.DATABASE_URL;
  if (url?.startsWith("mysql://")) {
    return mysql.createPool({ uri: url, connectionLimit: 10, waitForConnections: true });
  }
  return mysql.createPool({
    host: process.env.DB_HOST,
    port: Number(process.env.DB_PORT ?? 3306),
    user: process.env.DB_USER,
    password: process.env.DB_PASSWORD,
    database: process.env.DB_NAME,
    connectionLimit: 10,
    waitForConnections: true,
  });
}

const pool = global.__mysqlPool ?? buildPool();
if (process.env.NODE_ENV !== "production") global.__mysqlPool = pool;

export const db = drizzle(pool, { schema, mode: "default" });
export { pool };
```

3. Create `src/lib/db/schema.ts` using MySQL column builders. IDs use `varchar(36)` (Better Auth generates string IDs; application tables use `UUID()` default).

Better Auth tables (per https://better-auth.com/docs/adapters/drizzle — MySQL variant):
- `user`: id varchar(36) PK, name varchar(255), email varchar(255) unique, emailVerified boolean default false, image text, createdAt/updatedAt timestamp, role varchar(32) default "customer", banned boolean, banReason text, banExpires timestamp, pdpaConsentAt timestamp nullable (D-09)
- `session`: id varchar(36) PK, expiresAt timestamp, token varchar(255) unique, ipAddress varchar(45), userAgent text, userId varchar(36) FK -> user onDelete cascade
- `account`: id varchar(36) PK, accountId varchar(255), providerId varchar(64), userId varchar(36) FK, accessToken/refreshToken/idToken text, accessTokenExpiresAt/refreshTokenExpiresAt timestamp, scope text, password text
- `verification`: id varchar(36) PK, identifier varchar(255), value text, expiresAt timestamp

Application tables:
- `categories`: id varchar(36) PK default `(UUID())`, name varchar(100) unique, slug varchar(120) unique, createdAt timestamp
- `products`: id varchar(36) PK default `(UUID())`, name varchar(200), slug varchar(220) unique, description text, images JSON (array of relative upload URLs, max 5 enforced at app level per D-03), materialType varchar(64), estimatedProductionDays int, isActive boolean default true (ADM-04), isFeatured boolean default false (D-12), categoryId varchar(36) FK -> categories, createdAt/updatedAt timestamp (updatedAt uses `.onUpdateNow()`)
- `productVariants`: id varchar(36) PK default `(UUID())`, productId varchar(36) FK -> products onDelete cascade, size mysqlEnum ["S","M","L"] (D-13), price decimal(10,2) (MYR), widthCm/heightCm/depthCm decimal(6,1) nullable

Drizzle `relations()`:
- products -> category (one), products -> variants (many)
- productVariants -> product (one)
- categories -> products (many)
- user -> sessions/accounts (many)

CRITICAL: MySQL has no native array type — `images` is stored as JSON. Use `json("images").$type<string[]>().notNull().default([])`. This matters when filtering or reading from Drizzle.

4. Create `src/lib/mailer.ts` — nodemailer transport singleton + helpers:
```typescript
import nodemailer, { type Transporter } from "nodemailer";

declare global { var __mailTransport: Transporter | undefined; }

function buildTransport(): Transporter {
  const port = Number(process.env.SMTP_PORT ?? 587);
  return nodemailer.createTransport({
    host: process.env.SMTP_HOST,
    port,
    secure: port === 465,
    auth: process.env.SMTP_USER && process.env.SMTP_PASSWORD
      ? { user: process.env.SMTP_USER, pass: process.env.SMTP_PASSWORD }
      : undefined,
  });
}

export function getMailer() {
  if (!global.__mailTransport) global.__mailTransport = buildTransport();
  return global.__mailTransport;
}

export const MAIL_FROM = process.env.SMTP_FROM ?? "3D Ninjaz <noreply@3dninjaz.com>";

export async function sendResetPasswordEmail({ to, name, url }: { to: string; name: string; url: string }) {
  try {
    await getMailer().sendMail({
      from: MAIL_FROM,
      to,
      subject: "Reset your 3D Ninjaz password",
      html: `<p>Hi ${name},</p><p>Click <a href="${url}">here</a> to reset your password. This link expires in 1 hour.</p><p>If you did not request this, ignore this email.</p>`,
    });
  } catch (err) {
    console.error("[mailer] reset email failed:", err);
  }
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client -> Auth API | Untrusted signup/signin requests cross to Better Auth handler |
| Auth -> Database | Session/user data written to cPanel MySQL |
| App -> SMTP | Transactional emails sent via authenticated cPanel SMTP |
| Admin seed script | Elevated privilege operation creating admin user |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Spoofing | Auth API route | mitigate | Better Auth handles password hashing, session tokens, and cookie security. HTTP-only cookies prevent XSS token theft. |
| T-01-02 | Tampering | .env.local | mitigate | .env.local in .gitignore, never committed. Placeholder values in template only. |
| T-01-03 | Information Disclosure | DATABASE_URL | mitigate | Connection string only in .env.local (gitignored). cPanel Remote MySQL is IP-restricted to developer machines; production Vercel deploy should use the cPanel server's public IP whitelist OR tunnel via the same hosting account. |
| T-01-04 | Elevation of Privilege | User role field | mitigate | Role defaults to "customer" in both Better Auth config and DB schema. Admin only assignable via seed script, not registration API. |
| T-01-05 | Repudiation | PDPA consent | mitigate | pdpaConsentAt stored as server-side timestamp, not client boolean. Cannot be forged or backdated. |
| T-01-06 | Information Disclosure | SMTP credentials | mitigate | SMTP_USER/PASSWORD only in .env.local. TLS enforced on port 587 (STARTTLS) per nodemailer config. |
</threat_model>
