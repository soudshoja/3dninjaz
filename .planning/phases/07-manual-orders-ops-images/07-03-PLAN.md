## Trust Boundaries

|----------|-------------|
| Admin -> manual-order-form | Admin authenticated; form values still server-validated |

## STRIDE Threat Register
| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-03-money | Tampering | createPaymentLinkPayPalOrder | mitigate | Client posts ONLY `{token}`; server reads totalAmount from orders row. Test verify ensures no `amount` prop read in island. |
| T-07-03-PII-on-payment-link | Information Disclosure | /payment-links/<token> URL | mitigate | URL contains only random 32-char token; page renders item name + description + amount only — NO email/phone/address shown. |
| T-07-03-token-bruteforce | Spoofing | token guessing | mitigate | 192-bit entropy (`crypto.randomBytes(24).toString("base64url")`); brute-force infeasible. Optional follow-up: add rate-limit on /payment-links/* if abuse detected. |
| T-07-03-replay | Tampering | repeated capture call | mitigate | usedAt + paypalCaptureId both checked before capture; second call returns idempotent success or "already paid" 410. |
| T-07-03-EoP | Elevation of Privilege | createManualOrder | mitigate | requireAdmin() first await on every export. |
| T-07-03-XSS | Tampering | customItemDescription render | mitigate | Render via plain text node (using {variable} JSX expression — React auto-escapes). Never use raw HTML insertion APIs. Description capped 2000 chars by Zod. |
</threat_model>
