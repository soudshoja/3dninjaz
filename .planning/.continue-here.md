## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| Stale-closure in onSaved/restore | `await refetch()` then read `fields` (closure) — React state update is async; closure is stale until re-render | blocking | Pull fresh data via direct fetch (`getConfiguratorData(id)`) inside the same async block. Pattern fixed in a2f5e8c, 9ddef01. |
| Auto-hydrate-on-mount draft restore | If draft is auto-applied on mount, can wipe existing variant data with stale snapshot | blocking | ALWAYS banner-only Restore; explicit user click required. Ref: variant-editor.tsx, configurator-builder.tsx, inline-fields-editor.tsx, tier-table-editor.tsx. |
| Sub-agent leaves uncommitted edits | Agents edit a dependency file but stage only the "main" change → CI green locally, fails on push | blocking | After every agent return: `git status --short` + `git diff --stat` + verify CI run. Catch the dependency file before push. |
| Customer/admin scope confusion | Wiring autofill or autosave on customer PDP when intent was admin editor | blocking | Default assumption: admin. Confirm before customer-facing changes. |
| Over-engineering vs minimum viable | Bundling P1/P2/P3 audit findings into a single delivery when user asked only for P0 | advisory | Cap scope explicitly in Sonnet briefs ("ONE file, NO list, NO TTL"); defer extras to todos. See `feedback_simple_solutions_first.md`. |
